# Mock IPL Auction

Realtime IPL auction simulator built with Next.js, TypeScript, and Supabase.

This project is designed for college events and mock auction sessions where one admin runs the room and team captains bid live from their own consoles.

## What It Includes

- Role based login (`admin` and `team`)
- Admin auction controls:
  - nominate player
  - place bid
  - update timer and increment
  - sell player
  - mark unsold
  - reset auction
- Team console with live bid updates
- Team squad view with purse tracking
- Admin dashboard for sales and purse summary
- Slide manager for full screen announcements on team consoles
- CSV import from `IPL AUCTION DATA SHEET.csv`
- Realtime refresh through Supabase broadcast channels

## Tech Stack

- Next.js 16 (App Router)
- React 19
- TypeScript
- Tailwind CSS 4
- Supabase Auth
- Supabase Postgres
- Supabase Realtime Broadcast

## Project Structure

- `app/(auth)` login routes
- `app/admin` admin control room pages
- `app/team` team-facing pages
- `app/actions` server actions for auth, auction, players, and slides
- `components` shared UI and feature components
- `lib` Supabase clients, auth utilities, data loaders, CSV parsing
- `scripts` one-time setup and import scripts
- `supabase/migrations` database schema and RLS policies

## Prerequisites

- Node.js 20+
- npm
- A Supabase project

## Environment Variables

Copy `.env.example` to `.env.local` and fill in values:

```env
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=
SUPABASE_PROJECT_REF=
AUCTION_ADMIN_EMAIL=
AUCTION_ADMIN_PASSWORD=
TEAM_EMAIL_DOMAIN=
TEAM_PASSWORD_PREFIX=
UPSTASH_REDIS_REST_URL=
UPSTASH_REDIS_REST_TOKEN=
AUCTION_JWT_SECRET=
```

### Notes

- `SUPABASE_SERVICE_ROLE_KEY` is server-only and should never be exposed publicly.
- `TEAM_EMAIL_DOMAIN` and `TEAM_PASSWORD_PREFIX` are optional.
- `UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN`, and `AUCTION_JWT_SECRET` are optional unless you want the lower-latency Redis event transport.

## Local Setup

1. Install dependencies.

```powershell
npm install
```

2. Run the database migration in Supabase SQL Editor using:
   [supabase/migrations/001_initial.sql](./supabase/migrations/001_initial.sql)

3. Bootstrap admin and team accounts.

```powershell
npm run setup:auction
```

4. Import players from CSV.

```powershell
npm run import:players
```

5. Start the app.

```powershell
npm run dev
```

6. Open:
   `http://127.0.0.1:3000/login`

## Default Access Pattern

- Admin email comes from `AUCTION_ADMIN_EMAIL`
- Admin password comes from `AUCTION_ADMIN_PASSWORD`
- Team accounts are auto-generated by `npm run setup:auction`
- Generated team credentials are saved in `team_credentials` table

## Event Day Flow

1. Admin logs in and opens `/admin/players` to confirm pool size.
2. Admin goes to `/admin/auction` and nominates players.
3. Team captains log in on `/team/auction` and bid live.
4. Admin sells or marks unsold.
5. Dashboard and squad views update after every action.
6. Admin can push announcements from `/admin/slides`.

## Optional Lower-Latency Transport

The app now supports an optional Redis-backed event stream for faster UI refresh propagation while keeping Supabase as the source of truth.

- Auth stays on Supabase.
- Database writes stay on Supabase.
- Redis is used only to publish auction events quickly.
- The browser receives those events through an authenticated SSE route using a short-lived JWT.

See [REDIS_JWT_ROLLOUT.md](./REDIS_JWT_ROLLOUT.md) for the exact setup and requirements.

## Deployment (Vercel + Supabase)

This app is ready to deploy on Vercel with Supabase as backend.

1. Import the repo into Vercel.
2. Add the same environment variables in Vercel project settings.
3. Deploy.
4. Run `setup:auction` and `import:players` once against your production Supabase project.

## Troubleshooting

- Build issues from stale cache:
  - delete `.next`
  - run `npm run build` again
- Login works but team data is missing:
  - confirm `teams.user_id` is linked to auth users
  - rerun `npm run setup:auction`
- No players visible:
  - run `npm run import:players`
- Realtime feels stale:
  - verify Supabase env vars are correct
  - confirm migration was applied with realtime triggers
